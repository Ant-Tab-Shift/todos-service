# Техническое задание

## Задача

Написать HTTP-сервер на Go, который будет обрабатывать следующие эндпоинты:
* POST /todos — создать новую задачу
* GET /todos — получить список всех задач
* GET /todos/{id} — получить задачу по идентификатору
* PUT /todos/{id} — обновить задачу по идентификатору
* DELETE /todos/{id} — удалить задачу по идентификатору

## Сущность Task

Структура задачи должна включать:
- числовой идентификатор
- заголовок
- описание
- признак завершённости

## Требования

- Хранение данных без использования внешних хранилищ
- Логирование запросов
- Использование context для таймаутов/отмены
- Dockerfile для запуска приложения

### Валидация

При создании и обновлении задачи заголовок не должен быть пустым. Если валидация не прошла — вернуть статус 400 Bad Request.

### Обработка ошибок

Если задача с указанным идентификатором не найдена — вернуть 404 Not Found.

### Тесты

Напишите unit-тесты. Убедитесь, что обрабатываются основные кейсы: успешное создание, ошибка валидации, дублирование идентификатора

## Оценивание

- Качество кода (читаемость, идиоматичность Go, организация проекта)
- Обработка ошибок и крайних случаев
- Наличие и качество тестов
- Следование требованиям
